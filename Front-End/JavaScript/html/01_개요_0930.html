자바 스크립트

Hyper Text Markup Language(HTML)에서 기능 처리를 담당하는 언어.
사용자의 어떠한 행위들(클릭, 키보드 입력 등)을 이벤트라고 하는데, 이벤트가 발생했을 때, 동작하는 기능 처리를 담당한다.
- 컴파일 방식 : 전체 코드를 한꺼번에 읽어들여, 기계어로 변환하고 실행한다.
* 미리 전체 소스 코드를 변환한 뒤, 실행하기 때문에 수행속도가 빠름.
- 인터프리터 방식 : 코드를 한줄씩 읽어가며, 기계어로 변환하고 바로 실행하는 방식
* 코드를 한줄씩 읽고 변환하고 실행하므로, 상대적으로 컴파일 방식의 언어보다 수행속도가 느릴 수 있음.
* 한줄씩 번역 및 실행하므로, 실제 실행 시점에 오류가 발생.
- 자바는 컴파일 방식 + 인터프리터 방식의 특성을 모두 가지고 있어 하이브리드 언어라고 얘기한다.

자바 스크립트 사용법

1. inline(인라인) : 태그내에 소스코드를 작성하여 실행시키는 방식
2. internal(내부) : HTML 문서 내에, script 태그를 이용하여, 소스코드를 작성하여 실행시키는 방식
3. external(외부) : 별도의 .js 확장자를 가진 파일로 소스코드를 작성하여 실행시키는 방식
